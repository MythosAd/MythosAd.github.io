github 博客搭建简单指南

浪费了一天的时间搭建了blog,写了第一篇blog。假如不把搭建过程记录下来。

主要参考

https://zhangslob.github.io/2017/02/28/%E6%95%99%E4%BD%A0%E5%85%8D%E8%B4%B9%E6%90%AD%E5%BB%BA%E4%B8%AA%E4%BA%BA%E5%8D%9A%E5%AE%A2%EF%BC%8CHexo-Github/

前言

github本身支持html格式展示。建立仓库时将 Repository name 设置 用户名.github.io 即可达成 https://用户名.github.io/

环境搭配原理-主要利用hexo将发布blog打包成html。且hexo自身脚手架cli命令集合npm,git发布功能。并且支持markdown文本本地预览。

生产blog环境需要

1，申请github账户,建立仓库-用户名.github.io

2,安装 git,node,hexo 注意三者安装都=默认配置环境命令-path

3,安装git之后 使用ssh命令 生成ssh key，配置在github的key中

4，使用 hexo init [博客名] 建立第一个hex博客目录

5，进入source_posts 目录编写第一片博客。

编写博客ide我使用的是 markdownpad2，它使用了渲染引擎 Webkit 并且和npm一样 内置了JS引擎V8。

好处是，在livePreview可以左右观看实时效果。

6，编写完以后，使用hexo generate 命令生成deploy_git目录下html,js.css文件

7，使用hexo depoly可以直接发布至个人github。 或者使用hexo server 启动本地node.js服务查看预览效果。-访问链接localhost:4000

注意坑点

1，博客格式一定要正确,错误的文本符会导致generate报错，由于js解析报错，没有详细日志，无法查找原因。 例如我本人因为链接后缀”%}”写错，generate报错

2，我是win10的系统。使用markdownpad2，livePreview面板出错。好在有官方直接的帮助链接,安装一个解决兼容性的sdk包awesomium_v1.6.6_sdk_win，且升级intel内置显卡驱动。问题得到解决

备注

接触hex语法，有点类似emacs org-mode写东西。 不过最近肯定没什么时间弄了..以后再来弄
